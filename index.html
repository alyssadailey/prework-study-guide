HTML

  • The head element contains information about the webpage.
  • The body element represents the visable content shown to the user.
  • Elements included within the head element are not visible when a user looks at a webpage
  • Elements included within the body element are visible to users
  • The declaration tells the browser what document type it should expect when it is opened. This particular declaration is expecting an HTML on the page
  • !DOCTYPE is not case-sensitive because it's a dec. Decs are not case sensitive; HTML elements ARE case sensitive
  • The title element defines the title for a webpage. This element is required and is extremely important for SEO (search engine optimization)
  • SEO is the process of improving the ranking of a website in search engines such as google or bing. Better SEO results in your website means more frequently showing up in searches
  • The style element defines the CSS styles associated with a website
  • The base element defines the base URL for a webpage
  • The link element connects an external resource to the HTML document
  • The meta element defines metadata such as the character set, description, keywords, author and viewport
  • The script element embeds client-side scripts in the HTML document
  • elements that are nested inside other elements are called child elements
  • The body element is an example of a semantic HTML element
  • In semantic HTML elements the name of the element describes it's content and the role it plays on a webpage
  • It's important to use Semantic HTML elements because they make your HTML easier to read by other developers and the browser. They also play an important role in making sure your webpage is accessible to all, including those using assistive tech.
  • Not all elements are semantic. It's common to see non-semantic elements such as div to create a division in the page or spam to create an inline division
  • An element refers to the entire element from opening tag to closing tag. A tag refers only to what appears inside the angle brackets
  • h1,h2,h3,h4,h5,h6- these are levels of heading- always need to be in cronological order- h1 sould only be used one time per page
  • The p element represents a paragraph or block of text. You will use this extensively to wrap most text on your webpages
  • ul, ol, and li elements represent unordered lists, ordered lists, and list items
  • img element contains info about images that are displayed on the webpage. The src attribute is the most important attribute listed cause it defines the location or source of the image
  • br element creates a line of empty space, or a line break
  • a element stands for anchor. It creates links to the same webpage or other webpages
  • The href attribute points to the URL for the link
  • The alt attribute contains a text string that describes the appearance and functionality of the image
  • The alt text is important for screen readers and other assistives techs. Alt text can also improve search engine ranks for your website
  • The footer element appears at the bottom of the page and usually contains the author, copyright, contact, site map, and navigation

CSS

  • Cascading Style Sheets (CSS) is a programming language to define how webpage content is presented to the user.
  • A margin indecates how much space we want around the outside of an element. The margin property acts much lie a margin in a word processing doc.
  • A padding indicates how much space we want around the content inside an element. When you increase padding, you are actually making the entire element bigger. That is because the padding while transparent, is located inside the element, whereas the margin is the transparent buffer outside element.
  • There are 3 ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • When you want to style an element directly in the HTML file, they use inline CSS.
  • Ex. of inline CSS is using the style attribute to change the color of a single h1 heading.
  • A single change to the CSS rule will apply to all tagged page elements.
  • The internal style sheet contains CSS rules for a webpage in the head section of the HTML file.
  • Although inline CSS and internal style sheets are acceptable, the most common way to style a webpage is by using an external CSS style sheet, which allows developers to keep all of their CSS rules in a seperate file.
  • When using an external style sheet, we link the file with changes using a link element placed in the HTML head. In this ex we put it after the meta and before the title.
  • Seperation of concerns- widely used term that refers to seperating code into different files based on use.
  • To create a new feature branch use following git commands: git status, git checkout main, git checkout -b feature/add-css.
  • When breaking down a CSS rule, there is first a selector that defines the element or attribute to which the tules or declarations will apply
  • Declarations contain 2 important components: the CSS property we want to apply, and the value of said property
  • The CSS Box model teaches about content box, padding, border, and margin and how they interact with one another
  • Content box: the area where your content is displayed; size it using properties like inline-size and block-size or width and height
  • Padding box: the padding sits around the content as white space; size it using padding and related properties
  • Border box: the border box wraps the content and any padding; size it using border and related properties
  • Margin box: the outermost layer, wrapping the content, padding and border as whitespace between this box and other elements; size it using margin and related properties
  • In order to apply a rule to all the elements visible on the page use "*"
  • You can combine 2 selectors into a single CSS rule with a comma, allowing the declaration to affect both selectors
  • DRY- Don't Repeat Yourself- coding principle that encourages develpers to combine steps in order to reduce the number of lines used in code
  • The img selector assigns property values to any image element in the markup
  • The display property is assigned with the block value which assigns each image to start on a new line
  • Must format lists and p text in order to center all text and size the font
  • In order to link new changes, go back to main file, and add link to head
  • Used link rel="stylesheet" href="./assets/style.css" to add our changes
  • Link- a link element creates the connection to the external file
  • The rel attribute specifies the relationship between the current doc and the linked doc resource
  • The href attribute specifies the location (URL) of the external resource. The href attribute is using a relative path. This means the location is derived from the current location (in this case index.html). To locate the style sheet from the current location we must look in the assets, hence the assets/style.css
  • Class attributes will allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector
  • To differenciate topics, we added box-shadows around each note topics, to keep it visually appealing as well as organized
  • To add shadow boxes we used ".card" to apply the box to all topics
  • When selecting colors for elements of your webpage, we use an HTML hexadecimal code to define each color. This just means that there is a corosponding color for each and every specific shade
